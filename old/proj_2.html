BomberMan

This project was an assignment for INFO1113, which was developed individually. I had taken this unit in my second semester of 2021, as a result I may not remember most of my design choices but I will still try my best to explain my thoughts. Although it's an old project, I'm still deciding to display it as I am proud of this assignment. I believe I scored around 95% only losing marks for testing.

The goal of the game is to reach the goal tile of each level without running out of time or lives.



The app first reads the input, which are text files for the map.
P = Player
G = Goal
R = Red Enemy (different behaviour than the yellow)
Y = Yellow Enemy
W = wall



The user is able to modify the number of lives, time and levels.



The player is able move and drop bombs, which can kill and break. The shape and size of the explosion also change depending on where they are dropped.



The map is stored in an ArrayList due to levels having different sizes. I made an abstract class called GameObject which holds variables like coordinates, width and height. Everything extends from GameObject class as they all require positioning and everything being displayed has width and height. Tiles and People (characters) also have abstract classes. Tiles all share similar properties such as being breakable, empty or finish tile and people have properties such as movement and direction (for animation).
